系统安装

购买

       您可以直接和我们联系,或者选择我们合法授权的经销商购买 统一资源平台 的许可。

  • 单一应用许可

           使用或者基于Tekinfo统一资源平台开发的应用系统，只能部署在一个安装应用。

  • 机构多应用许可

           在一个机构中，可以部署在多个安装应用。

  • 扩展许可

           在一个机构中，可以部署在多个安装应用。同时允许改机构发布包含 统一资源平台 的扩展应用。但不允许直接发布tekinfo 统一资源平台本身。
    本应用主要支持第三方开发者，基于 统一资源平台 开发用户实际应用系统并发布。

  • J2EE Servlet

           根据您部署的业务环境，选择您的Servlet容器。一个推荐的低成本选择是下载Tomcat 免费开源的Servlet容器。

       请参考： 统一资源平台 的许可声明。

准备
  • 购买许可

           请购买 统一资源平台 的合法授权版本。同时检查该授权的使用是否在合法范围内。

  • 操作系统

           统一资源平台 支持Windows (建议采用服务器版本)及Unix/Linux等系统。在准备安装和部署 统一资源平台 前，请确定你将要使用的操作系统以及相关配置的硬件。

  • 数据库

           请选定您要使用的数据库，并且提前安装好数据库系统。检查 统一资源平台 的config/database/driver 下是否有该数据库的驱动定义以及定义中的jdbc库文件。如果缺失，请像您的经销商索取。

  • JDK

           通过JAVA官方网站获取可靠的JDK版本下载。下载前请确认您购买的 统一资源平台 版本所需要支持的JDK版本。

  • J2EE Servlet

           根据您部署的业务环境，选择您的Servlet容器。一个推荐的低成本选择是下载Tomcat 免费开源的Servlet容器。

安装

       统一资源平台 安装非常简单，只要把程序包拷贝到您配置好的Servlet容器中完成。下面以windows下安装Tomcat中的统一资源平台为例说明：

  1. 环境变量
    在操作系统中成功安装所需JDK后，设置“环境变量”（在Windows系统中，右键点击“我的电脑”->"属性"->"高级系统设置"->"环境变量",在系统变量中添加）：
    JAVA_HOME= <jdk实际安装路径>;
    PATH=<补充jdk实际安装路径/bin>;
  2. Tomcat
    下载并安装Tomcat。根据需要，在Tomcat安装路径的conf目录下，配置启动端口。（建议使用SSL端口，提高安全性）
  3. 复制Tekinfo
    将Tekinfo程序包复制到tomcat项目路径下，可以使用webapps/tekinfo目录，也可以直接覆盖webapps/ROOT。具体参看tomcat文档。
  4. 配置数据库
    在Tekinfo程序包目录下，找到：
    config/database/driver
    下，使用的数据库配置文件，例如:config/database/driver/mysql/mysql.xml。使用文本编辑软件编辑：
    <url value="jdbc:mysql://localhost:3306/<您创建的数据库名>" />
    <user value="<数据库连接用户名>" />
    <password value="数据库连接用户保密字" />
  5. 启动
    配置完成，启动tomcat即可。
初始化

       统一资源平台 安装完成后，需要对数据库进行初始化。步骤如下：

  1. 运行
    首先运行安装有统一资源平台的Servlet容器。成功运行后，使用浏览器（推荐Firefox/Chrome）访问相应的端口和你配置的tekinfo路径。例如：
    http://localhost:8080/tekinfo
    或者
    http://localhost:8080/
  2. 管理员控制台
    登录后访问管理员控制台，或者直接访问控制台按照提示登录。初始安装后：
    用户名：admin
    保密字：tekinfo
    对于使用 统一认证 作为登录系统的用户，相关配置请访问您购买的ipass系统或者访问http://ipass.space寻求帮助。
  3. 初始化数据库
    在管理员控制面板，选择“初始化数据库”选项，对数据库系统进行初始化。数据库初始化主要是创建相关数据库表。
  4. 初始化对象
    在管理员控制面板，选择“初始化对象”，点击运行，系统将把运行所需要的初始对象导入到数据库表中。
  5. 重新启动
    设置完成后，重新启动系统即可。
